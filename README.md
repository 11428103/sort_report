# 氣泡排序法

## 1. 簡述原理

每次比較相鄰的兩個元素，如果大的數字在前就交換。每一輪過後，最大的元素會像「氣泡」一樣浮到數列的最右端<br>

## 2.執行步驟

1.陣列開頭開始，比較相鄰的兩個數字  
2. 如果左邊比右邊大，就交換位置；否則不動。  
3. 移動到下一對（第 2 個和第 3 個），重複比較與交換  
4. 第一輪結束後，最大的數字會被擠到最後面  
5.針對剩下的 n - 1 個數字重複上述動作，直到沒有任何數字需要交換<br>


## 3. 複雜度分析

平均 / 最差: $O(n^{2})$

最佳：O(n)（若數列已排好且有設置旗標偵測）<br>

# 選擇排序法

## 1. 簡述原理

在未排序的部分中尋找最小值，將其與未排序部分的第一個元素

交換位置。簡單來說，就是「每次都挑最小的出來放好」<br>


## 2.執行步驟

1. 假設第一個位置是最小的
2. 從剩下的未排序數字中，掃描一遍找出真正的「最小值」  
3. 將這個真正的最小值與第一個位置的數字交換  
4. 第一輪結束後：第一個位置已經排好  
5. 從第二個位置開始，重複上述動作，直到倒數第二個數字處理完<br>


## 3.複雜度分析

平均 / 最差 / 最佳： $\mathrm{O}(n^{2})$ （不論初始狀態為何，都必須掃描剩下的元素）<br>

# 插入排序法

## 1. 簡述原理

將未排序的數字逐一取出，在已排序的序列中「從後往前掃描」，找到

對應位置後插入<br>


## 2.執行步驟

1. 把第一個數字視為「已排序」  
2. 取出下一個數字 (未排序), 作為「待插入項」  
3. 將待插入項與「已排序部分」從右向左比較  
4. 如果已排序的數字比待插入項大，就將該數字向右移一格  
5. 找到適合的位置後，將待插入項放下  
6. 重複步驟 2-5，直到所有數字都進入已排序部分<br>


## 3. 複雜度分析

平均 / 最差: $\mathrm{O}\left(n^{2}\right)$

最佳：O(n) (當資料已接近排序完成時，效率極高)<br>

# 合併排序法

## 1. 簡述原理

採用分治法 (Divide and Conquer)，將陣列不斷對半拆分，直到每個小塊只有一個元素（視為已排序），接著再將這些小塊兩兩「合併」成更大的排序數列<br>


## 2.執行步驟

1. 分割 (Divide)：將陣列從中間對半切開，直到每一組都只有 1 個數字（1 個數字自動視為已排序）

2. 合併 (Merge)：開始將兩個小的排序數列合併成一個大的

- 比較兩個數列的「頭」  
- 取出較小的那個放入暫存陣列  
- 重複直到兩個數列都取完

3. 遂迴執行以上動作，直到所有小組都合併回一個完整的排序陣列<br>


## 3. 複雜度分析

平均 / 最差 / 最佳: $O(n \log n)$<br>

# 快速排序法

## 1. 簡述原理

採用分治法，選定一個基準點 (Pivot)，將數列分為「比基準點小」與「比基準點大」兩部分，再對這兩部分遞迴執行相同操作<br>


## 2.執行步驟

1. 挑選基準 (Pivot)：從陣列中選一個數字（通常選第一個、最後一個或中間）

2. 分區 (Partition) :

- 把所有小於基準點的數字移到左邊  
- 把所有大於基準點的數字移到右邊  
- 基準點最後放在中間位置

3. 邐迴：對左邊和右邊的兩個小陣列重複上述「挑選基準」與「分區」的動作  
4. 當子陣列的大小縮小到 0 或 1 時，排序就完成了<br>


## 3. 複雜度分析

平均 / 最佳: $O(\log n)$ (實務上通常是最快的排序法)

最差: $O\left(n^{2}\right)$<br>

# 學習心得

在執行複雜度分析的過程中，我發現有一個點特別有趣，那就是合併排序法的整體速度竟然比「一般稱為最快速」的快速排序法還要快，這點讓我覺得很奇怪，最後查詢資料才發現，我的程式碼是建立在「完全逆序」的狀況下，而快速排序法在執行「完全逆序+選最後一項為基準（Pivot）」的資料的複雜度會從 $O(n\log n)$ 变成 $O(n^{2})$ ，一般來說基準是會選數列的中位數，如果選了最後一位數字當中位數那將造成遞迴呼叫太多次，使得速度變慢，因此我從這次的實驗中挖掘出快速排序法在最不利的情況，並不是在每個條件下都是最快的
